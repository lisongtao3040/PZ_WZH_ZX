# FileServ2.aspx 功能升级说明

## 更新日期

## 新增功能

### 1. 图片搜索功能（合并双源）
**功能描述：**
- **合并搜索**：一次搜索同时从文件服务器和数据库检索
- **分类标识**：通过 `[数据库]` 标签清晰区分数据来源
- 支持按文件名关键词模糊搜索，不区分大小写
- 提供"清除搜索"按钮快速恢复文件夹浏览模式
- 显示搜索结果统计（文件夹项目数和数据库图片数）

**使用方法：**
1. 在搜索框中输入图片名称或关键词
2. 点击"搜索"按钮
3. 系统会同时从两个来源搜索：
   - 共享文件夹中的文件和文件夹
   - 数据库 `m_picture_km` 表中的图片
4. 结果合并显示在一个列表中
5. 数据库中的图片会显示 `[数据库]` 橙色标签
6. 顶部显示统计信息："找到 X 个文件夹项目，Y 个数据库图片"
7. 点击"清除搜索"按钮可返回共享文件夹根目录

**技术实现：**
- 前端：`txtSearch` 文本框 + `btnSearch` / `btnClearSearch` 按钮
- 后端：`SearchKeyword` ViewState 属性保存搜索状态
- 新增 `BindGridMerged()` 方法：
  - 先从文件系统搜索（Try-Catch 保护）
  - 再从数据库搜索（Try-Catch 保护）
  - 合并结果到一个列表
  - 统计并显示两类结果数量
- 过滤逻辑：
  - 文件系统：使用 `IndexOf` 进行不区分大小写的匹配
  - 数据库：调用 `ComDDL.SelChkPictureByName()` 查询 `m_picture_km` 表

### 2. 批量下载功能
**功能描述：**
- 支持同时选择多个文件或文件夹进行打包下载
- 自动将选中的项目压缩为 ZIP 格式
- 保持原有文件夹结构
- 文件名包含时间戳，便于识别

**使用方法：**
1. 勾选要下载的文件或文件夹前的复选框
2. 点击"批量下载选中"按钮
3. 系统会自动打包并下载 ZIP 文件
4. 下载文件名格式：`批量下载_yyyyMMdd_HHmmss.zip`

**技术实现：**
- 使用 Ionic.Zip 库进行压缩
- 支持 UTF-8 编码，正确处理中文文件名
- 文件夹会保持原有目录结构
- 单个文件直接添加到 ZIP 根目录

---

### 3. 缩略图显示功能
**功能描述：**
- 自动识别图片文件（jpg、jpeg、png、gif、bmp、webp 等格式）
- 在 GridView 中显示 80x80 像素的缩略图
- 点击缩略图可查看大图预览
- 非图片文件显示文件图标 📄，文件夹显示文件夹图标 📁
- **支持数据库图片缩略图**：从数据库中读取图片数据并动态生成缩略图

**支持的图片格式：**
- JPG/JPEG
- PNG
- GIF
- BMP
- WebP
- SVG
- ICO

**使用方法：**
1. 上传图片后，图片文件会自动显示缩略图
2. 数据库搜索结果也会显示缩略图
3. 鼠标悬停在缩略图上会有放大效果
4. 点击缩略图可在遮罩层中查看完整尺寸的图片
5. 再次点击遮罩层或右上角 × 按钮关闭预览

**技术实现：**
- 共享文件夹图片：通过 `FileServ2.aspx?thumbnail=相对路径` 请求
- 数据库图片：通过 `FileServ2.aspx?dbthumb=picName|picUpdTime` 请求
- 高质量缩放：使用 HighQualityBicubic 插值算法
- 保持宽高比：自动计算最佳缩放比例
- 缓存友好：每次请求实时生成，可根据需要添加缓存机制

### 4. 数据库图片管理（只读）
**功能描述：**
- 从数据库 `m_picture_km` 表检索图片
- **新增“来源”列**：明确显示图片来源（数据库/文件夹）
  - 数据库来源：显示“数据库”（橙红色）
  - 文件夹来源：显示“文件夹”（绿色）
- **灰色行背景**：数据库来源的行显示灰色背景，便于视觉区分
- **数据库图片不能删除**：删除按钮自动隐藏
- **数据库图片不支持批量下载**：批量下载时自动跳过
- **批量删除按钮智能禁用**：选中数据库记录时，批量删除按钮自动禁用
- 可以单独下载数据库中的图片
- 显示图片更新时间信息

**数据来源：**
- 数据库表：`m_picture_km`
- 字段：`pic_name`（图片名）、`pic_upd_time`（更新时间）、`pic_conn`（图片二进制数据）

**使用方法：**
1. 点击“搜索”按钮搜索图片
2. 在“来源”列查看图片来源：
   - “数据库”（橙红色）= 来自数据库
   - “文件夹”（绿色）= 来自共享文件夹
3. 数据库图片的删除按钮不会显示
4. 点击下载按钮可单独下载数据库图片
5. 批量操作时数据库图片会被自动跳过

**技术实现：**
- 使用 `ComDDL.SelChkPictureByName()` 查询数据库
- 使用 `ComDDL.SelChkPictureKM()` 获取单个图片详情
- 通过 `SourceType` 字段区分数据来源（File/DB）
- GridView 新增“来源”列（TemplateField）
- RowDataBound 事件中根据 SourceType：
  - 设置来源文本和颜色
  - **设置行背景色（数据库=灰色，文件夹=默认/蓝色）**
  - 控制删除按钮可见性
- JavaScript 智能全选功能：
  - **全选时自动跳过数据库来源的行**
  - 实时检测选中状态，智能禁用批量删除按钮

## 注意事项

1. **性能考虑**：
   - 缩略图是实时生成的，如果图片数量很多，建议添加缓存机制
   - 批量下载大量文件时可能需要较长时间，请耐心等待
   - 数据库搜索限制为 top 100 条记录

2. **兼容性**：
   - 需要 .NET Framework 支持 System.Drawing 命名空间
   - Ionic.Zip 库必须已正确引用
   - 需要访问 `m_picture_km` 数据库表的权限

3. **安全性**：
   - 所有文件路径都经过验证，防止路径遍历攻击
   - 缩略图请求只允许访问配置的文件服务器路径
   - 数据库图片为只读模式，不能删除或修改

4. **数据来源区分**：
   - 共享文件夹图片：可以删除、批量下载
   - 数据库图片：只能单独下载，不能删除，不支持批量下载
   - 通过 `[数据库]` 标签清晰标识来源

5. **空值处理**：
   - 所有 DataKeys 访问都添加了空值检查
   - 使用 Try-Catch 捕获可能的异常
   - 提供默认值防止 NullReferenceException
   - 反射访问属性时也进行了空值判断

6. **后续优化建议**：
   - 可以添加缩略图缓存，减少重复生成
   - 可以支持更多图片格式（如 TIFF、RAW 等）
   - 可以添加分页功能以处理大量文件
   - 可以添加排序功能（按名称、日期、大小等）
   - 可以考虑将数据库图片也支持批量下载（分别打包）
