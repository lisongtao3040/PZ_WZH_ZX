# 拍照共有图片机能

## 機能概要

检查画面提供二维码 → 用户打开 iPhone 相机扫码或拍照扫码，根据二维码中 URL 转到 URL 上 → 用户在画面上再点拍照，照片保存到共有路径上。

## 実装ファイル構成

```
/PCWeb/
├── Inspection.aspx          # 検査画面 - QRコードを表示
├── Inspection.aspx.cs       # 検査画面 コードビハインド
├── PhotoUpload.aspx         # 写真アップロード画面 - 撮影・アップロード
├── PhotoUpload.aspx.cs      # 写真アップロード画面 コードビハインド（サーバー保存処理）
├── SharedPhotos/            # 写真保存先フォルダ（自動生成）
├── Web.config               # SharedPhotoPath 設定
```

## 処理フロー（実装済み）

### 1. 検査画面 → QRコード表示（Inspection.aspx）

- 检查画面提供二维码
- QRコードには `http://[host]/PhotoUpload.aspx` のURLが含まれる
- JavaScript (`qrcodejs`) を使用して動的にQRコードを生成

### 2. ユーザーがQRコードをスキャン

- 用戶打開 iPhone 相机扫码或拍照扫码
- 根据二维码中 URL 转到 PhotoUpload.aspx 上

### 3. 写真撮影・アップロード（PhotoUpload.aspx）

- **カメラ撮影**: ブラウザの `getUserMedia()` API を使用してリアルタイムカメラプレビュー → キャプチャ（JPEG）
- **アルバム選択**: `<input type="file">` を使用して端末内の画像を選択
- **撮り直し**: 撮影後もやり直し可能
- **プレビュー**: 撮影後すぐに画像を確認

### 4. 写真を共有パスに保存（PhotoUpload.aspx.cs）

- Base64エンコードされた画像データをサーバー側でデコード
- ファイル名: `PHOTO_yyyyMMdd_HHmmss_fff_xxxx.jpg`
- 保存先設定:
  - **デフォルト**: `~/SharedPhotos/`（アプリケーションルート配下）
  - **カスタム**: `Web.config` の `SharedPhotoPath` でネットワーク共有パスを指定可能
    - 例: `\\192.168.1.100\SharedPhotos\`
    - 例: `D:\SharedPhotos\`
- 保存後に保存済み写真一覧を表示（新しい順）

## Web.config 設定

```xml
<appSettings>
    <!-- 写真共有先のパス（ネットワーク共有パスまたはローカルパスを指定）
         例: \\192.168.1.100\SharedPhotos\  または D:\SharedPhotos\
         空の場合はアプリケーションルート配下の SharedPhotos フォルダを使用 -->
    <add key="SharedPhotoPath" value="" />
</appSettings>
```

## 補足

- .NET Framework 4.5.2 / ASP.NET Web Forms
- iPhone カメラスキャン対応（HTML5 + レスポンシブデザイン）
- 画像はJPEG形式で保存（品質85%）
- ファイル名はタイムスタンプ＋ランダム文字列で一意性を確保
