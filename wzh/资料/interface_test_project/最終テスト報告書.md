## 重要な発見

### 成功の鍵となる要素
1. **文字エンコーディング**: `Shift_JIS` エンコーディングを使用する必要があります
2. **Content-Type**: `application/x-www-form-urlencoded`
3. **データ形式**: JSON文字列を直接バイト配列に変換して送信
4. **フォームフィールド名不要**: フォームフィールドにラップせずに直接JSONデータを送信

### 成功したリクエスト設定
```python
# JSONデータ
json_string = json.dumps(test_data, ensure_ascii=False)

# Shift_JISエンコーディングを使用
encoded_data = json_string.encode('shift_jis')

# リクエストヘッダー
headers = {
    'Content-Type': 'application/x-www-form-urlencoded',
    'User-Agent': 'InterfaceTester/1.0'
}

# リクエスト送信
response = requests.post(url, data=encoded_data, headers=headers)
```

## 問題解決プロセス

### 初期の問題
- すべてのテストが500エラーを返す
- エラーメッセージ: "No JSON object could be ScanOnced"

### 根本原因
1. **文字エンコーディングの不一致**: インターフェースはShift_JISエンコーディングを期待しているが、UTF-8を使用していた
2. **データ形式の誤り**: フォームフィールドラッピングを試みたが、インターフェースは直接JSONデータを期待していた

### 解決策
C#コードの分析を通じて発見：
- `Shift_JIS` エンコーディングを使用
- `Content-Type: application/x-www-form-urlencoded`
- 直接JSONバイト配列を送信
